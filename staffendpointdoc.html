Endpoint: /api/staff/

Access staff information including details about specific staff within the system.

Available Endpoints

1. GET /api/staff/:id

Retrieve details of a specific staff by their ID.

  • Parameters:
    • id: user_id (string)
  • Response:
    • Success (200): Returns staff details.
    • Not Found (404): If staff with the given ID does not exist.
    • Invalid Data Type (400): If the provided ID is not a string.
    • Internal Server Error (500): If there's a server-side issue.

2. GET /api/staff/

Retrieve a list of all staff.

  • Query Parameters:
    • page: Page number for pagination (optional)
    • limit: Number of results per page (optional)
    • sort: Sort staffs by a specific field (optional)
  • Response:
    • Success (200): Returns a paginated list of teachers.
    • Internal Server Error (500): If there's a server-side issue.

3. GET api/staff/search?q=:query

Search for staff based on a search query.

  • Query Parameters:
    • q: Search query (string)
  • Response:
    • Success (200): Returns staffs matching the search query.
    • Bad Request (400): If the search query parameter is missing or invalid.
    • Internal Server Error (500): If there's a server-side issue.

Additional Information

The /staff endpoint provides access to staff information such as name, email, and associated department. For pagination, use the page and limit query parameters to navigate through the results. For searching, use the q query parameter to find teachers by name or email. Contact support for any issues related to teacher endpoints.
