API Documentation for Task Management

Root API endpoint is "api.bike-csecu.com/api/tasks/{action}

GET /api/tasks

Retrieves all tasks from the Assign_Task table.

Request Body

None

Query Parameters

  • page (optional): The page number for pagination. Default is 1.
  • limit (optional): The number of records per page. Default is 10.
  • sort (optional): The column to sort the results by. Default is 'task_id'.
  • order (optional): The order of sorting (asc or desc). Default is 'asc'.

Response

{
  "tasks": [
    {
      "task_id": 1,
      "assign_by": "user1",
      "assign_to": "user2",

GET API Endpoint: /api/tasks/mytasks

Retrieves tasks assigned to the logged-in user.

Request Body

None

Query Parameters

  • page (optional): The page number for pagination. Default is 1.
  • limit (optional): The number of records per page. Default is 10.
  • sort (optional): The column to sort the results by. Default is 'task_id'.
  • order (optional): The order of sorting (asc or desc). Default is 'asc'.

POST API Endpoint: /api/tasks/add

Creates a new task in the Assign_Task table.

Request Body

{
  "assign_to": "user2",
  "assign_date": "2022-01-01T00:00:00.000Z",
  "due_date": "2022-01-31T00:00:00.000Z",
  "task_title": "Task 1",
  "task_description": "Description of Task 1",
  "task_attachment": "attachment1.pdf",
  "task_status": 0
      }

Response

{
        "success": true,
        "message": "Task added successfully to Assign_Task table.",
        "form_id": 3
      }

PUT /api/tasks/update/:id

Updates an existing task in the Assign_Task table.

Request Body

Actually it will be used to update the status of that particular task. You don't need to pass all the fields. Pass those what field you want to update

{
  "assign_date": "2022-01-01T00:00:00.000Z",
  "due_date": "2022-01-31T00:00:00.000Z",
  "task_title": "Task 1",
  "task_description": "Description of Task 1",
  "task_attachment": "attachment1.pdf",
  "task_status": 0
      }

Response

{
  "message": "Task updated successfully"
      }

DELETE /api/tasks/delete/:id

Deletes an existing task from the Assign_Task table.

Request Body

None

Response

{
  "success": true,
  "message": "Task with id 3 deleted successfully"
      }

Note: The above documentation assumes that the user is authenticated and has the necessary permissions to access the API endpoints.
